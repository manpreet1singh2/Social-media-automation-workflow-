Node-by-Node Explanation
01
⏰ + 📊 Trigger → Google Sheets Read → Filter
TRIGGER + DATA ▼
Runs hourly. Reads the ContentQueue sheet filtering rows where Status=Ready. The Filter code node then checks if the BestTimeSlot matches the current hour — only due posts proceed.
// BestTimeSlot format: "14" (hour) or "2026-03-21 14" (date+hour) const currentHour = new Date().getHours(); const isDue = (!slot) || (scheduledDate === currentDay && scheduledHour === currentHour); if (!isDue) continue; // skip — not time yet

03
⚙️ Format Node — Parse AI + Build Per-Platform Posts
CODE NODE ▼
Takes the single AI response and explodes it into N items — one per platform. Each item has its full caption with hashtags appended. Passes all config metadata forward.
// One input → N outputs (one per platform) for (const platform of platforms) { const hashtagStr = hashtags.map(h => h.startsWith('#') ? h : `#${h}`).join(' '); const fullCaption = platform === 'twitter' ? caption // Twitter: no extra hashtags needed : `${caption}\n\n${hashtagStr}`.trim(); // → emits one item per platform }

05
🔴 Error Handling + Telegram Notifications
ERROR HANDLING ▼
Every API node has continueOnFail: true. The Collector normalizes success/fail. The IF node routes failures to Telegram error alert, successes to success alert.
// Error message sent to Telegram `🔴 *POST FAILED* 📍 Platform: ${platform} 📝 Topic: ${topic} ❌ Error: ${error} 🕐 Time: ${posted_at}` // Success message `✅ *POST PUBLISHED* 📍 Platform: ${platform} 🔗 URL: ${post_url}`
Error TypeHandling
API 401/403Log + Telegram alert (credentials expired)
API 429Retry 3× with exponential backoff
Network timeoutRetry 3×, then log as failed
AI parse errorFallback to raw content template
Empty contentSkip row, log warning

Setup Instructions (8 Steps)
1

Import Workflow

In n8n: Settings → Import Workflow → paste JSON. All 28 nodes will appear pre-wired across 3 sub-workflows.

2

Create Google Sheet

Create a sheet with 4 tabs: ContentQueue, PostLogs, Analytics, Leads. Add column headers from schema above.

3

OpenAI API Key

Add OpenAI credential in n8n Credentials. Model: gpt-4o for main, gpt-4o-mini for analytics/DM to save costs.

4

Platform Credentials

Add OAuth2 credentials for LinkedIn, Twitter, YouTube. For Instagram/Facebook, use n8n Variables to store Page Tokens.

5

Set n8n Variables

In n8n Settings → Variables: INSTAGRAM_BUSINESS_ID, INSTAGRAM_ACCESS_TOKEN, FACEBOOK_PAGE_ID, FACEBOOK_PAGE_TOKEN, TIKTOK_ACCESS_TOKEN, TELEGRAM_CHAT_ID

6

Update LinkedIn URN

In the LinkedIn node JSON body, replace YOUR_LINKEDIN_PERSON_ID with your profile URN. Get it from GET /v2/me API call.

7

Test with 1 Row

Add 1 row to ContentQueue: set Topic, RawContent, Platforms=linkedin, Status=Ready, leave BestTimeSlot empty (posts immediately).

8

Activate All 3 Schedules

Enable: ⏰ Schedule Trigger (Every Hour), ⏰ Analytics Trigger (Daily), ⏰ DM Monitor (Every 15min). Disable Manual Triggers in production.

Required API Integrations

📘 Meta / Facebook Graph API

  • Facebook Developer App with pages_manage_posts permission
  • Instagram Business Account linked to Facebook Page
  • Long-lived Page Access Token (60-day, auto-refresh)
  • Instagram Graph API for media upload + DM access
  • instagram_manage_messages permission for DM bot

💼 LinkedIn API

  • LinkedIn Developer App with w_member_social permission
  • OAuth 2.0 access token (60-day)
  • Author URN: call GET /v2/me to get person ID
  • For company pages: use organization URN instead
  • r_liteprofile + w_member_social scopes

🐦 Twitter / X API

  • Twitter Developer Account (Basic tier minimum)
  • OAuth 2.0 with PKCE for user auth
  • tweet.write + offline.access scopes
  • For media: use v1.1 media/upload endpoint first
  • Rate limit: 50 tweets/15min (Basic tier)

▶ YouTube Data API v3

  • Google Cloud Console project with YouTube Data API v3 enabled
  • OAuth 2.0 credentials (Web app type)
  • Scopes: youtube.upload + youtube
  • Video file must be accessible via public URL for upload
  • Shorts: video ≤60s, vertical 9:16, #Shorts in title/desc

🎵 TikTok Content Posting API

  • TikTok for Developers account + app creation
  • Content Posting API access (requires approval)
  • Scopes: video.publish + video.upload
  • Video URL must be publicly accessible (no auth required)
  • Server-to-server: use PULL_FROM_URL source type

🧠 OpenAI API

  • OpenAI API key (GPT-4o tier recommended)
  • Cost estimate: ~$0.02-0.05 per content package
  • Use GPT-4o-mini for analytics/DM replies (10× cheaper)
  • Set max_tokens: 2000 for main, 500 for quick replies
  • Temperature: 0.7 content, 0.4 analytics, 0.6 DMs
Scalable Architecture for 100K+ Followers
🔁 Content Cadence
Post 3-4x/week on LinkedIn, 1x/day Instagram, 3-5x/day Twitter, 2-3x/week Facebook. Set BestTimeSlot per platform in your queue sheet.
🖼 Media Pipeline
Use Cloudinary for auto-resizing. Store master images, generate platform-specific sizes (1:1 for IG, 16:9 for YT, 9:16 for TikTok/Reels) via Cloudinary transformations.
📊 A/B Testing
Add a "Variant" column to ContentQueue. Run 2 versions of a post. Compare engagement in Analytics sheet. Let AI pick winners automatically.
⚡ Queue System
Add a separate "ContentIdeas" sheet. Use a second workflow to auto-fill ContentQueue from RSS + AI ideas daily. Never run out of content.
🛡 Rate Limiting
Add a Wait node (2-5s) between platform posts. Keep batch size at 1 per run. Use n8n's built-in retry with exponential backoff (5s, 10s, 30s).
🔐 Token Refresh
Meta tokens expire in 60 days. Add a separate workflow that runs monthly to refresh tokens via OAuth refresh endpoint and update n8n Variables.
